# Design Package Structure Reference

This documents the standard file layout and purpose of every file inside a design system package generated by the Agent Orchestration Platform.

## Package Exports

All packages use the `@design/` npm scope and are installed as `file:` dependencies. The exports block in `package.json` always follows this shape:

```json
{
  "exports": {
    "./tokens.json":          "src/tokens.json",
    "./tokens.css":           "src/tokens.css",
    "./theme.css":            "src/theme.css",
    "./tailwind-preset.css":  "src/tailwind-preset.css",
    "./images/*":             "src/images/*",
    ".":                      "src/tokens.json"
  }
}
```

## File Purposes

| File | Import path | Purpose | When to use |
|------|-------------|---------|-------------|
| `src/tailwind-preset.css` | `@design/[pkg]/tailwind-preset.css` | Tailwind v4 `@theme` block mapping tokens to Tailwind utility classes | Always — first design import in main CSS |
| `src/theme.css` | `@design/[pkg]/theme.css` | Full theme: CSS `:root` custom properties + component-level CSS classes | Always — second design import in main CSS |
| `src/tokens.css` | `@design/[pkg]/tokens.css` | CSS `:root` custom properties only (subset of theme.css) | Only if you need tokens without component classes |
| `src/tokens.json` | `@design/[pkg]/tokens.json` | Structured token data for JS/TS access | When you need token values programmatically |
| `src/images/*` | `@design/[pkg]/images/[filename]` | Brand images (logos, wordmarks) | In Image components referencing brand assets |
| `src/components/*.html` | Read from filesystem | Component showcase HTML — visual/structural reference | Read before building any component category |
| `reference/pages/*.html` | Read from filesystem | Full page reference designs | Read to understand intended page layouts |

## CSS Import Order

The order of imports matters for Tailwind v4:

```css
/* 1. Tailwind base — must come first */
@import "tailwindcss";

/* 2. Design token → Tailwind utility mappings */
@import "@design/[package-name]/tailwind-preset.css";

/* 3. Full theme with component classes */
@import "@design/[package-name]/theme.css";
```

Never reverse these or merge into a single import.

## Token Access Patterns

### In Tailwind classes
Use the CSS variable name as a Tailwind arbitrary value:
```tsx
<div className="bg-[hsl(var(--primary))] text-[hsl(var(--primary-foreground))]">
```

Or, if the Tailwind preset maps them to utilities (check tailwind-preset.css):
```tsx
<div className="bg-primary text-primary-foreground">
```

### In inline CSS / CSS modules
```css
.my-element {
  background-color: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
  border: 1px solid hsl(var(--border));
}
```

### In JavaScript/TypeScript
```typescript
import tokens from "@design/[package-name]/tokens.json";

// Access a color's HSL value
const primaryHsl = tokens.colors.primary.hsl;  // e.g. "217 87% 17%"
const primaryCss = tokens.colors.primary.css;  // e.g. "hsl(217 87% 17%)"
```

## Component Showcase Files

Each `src/components/*.html` file demonstrates a category of components. Read the relevant file before building that category.

Common showcase files (names may vary by project):
- `form-elements.html` — inputs, selects, checkboxes, radio buttons, labels
- `navigation.html` — sidebar nav, breadcrumbs, tabs, top bar
- `layout.html` — page shell, content areas, cards, panels
- `data-display.html` — badges, stats, key-value displays, avatars
- `data-table--tanstack.html` — sortable/filterable tables with TanStack Table
- `charts--recharts.html` — bar, line, area, pie charts with Recharts
- `notifications---dialogs.html` — modals, alerts, confirmation dialogs
- `toast-notifications--sonner.html` — toast system with Sonner
- `command-palette--cmdk.html` — command palette / search overlay

## Reference Pages

`reference/pages/*.html` are full-page designs showing how components compose into real screens. Use them to understand intended layout, spacing, and hierarchy — not as copy-paste source.

Common reference pages (names may vary by project):
- `dashboard.html` — main dashboard with stats and charts
- `login-page.html` — authentication screen
- `user-management.html` — data table admin page
- Additional pages vary by project domain

## Hard-Coded Values Warning

The showcase and reference HTML files render standalone in a browser, so they may contain hard-coded `hsl(...)` or `rgba(...)` values in `style` attributes. These are for standalone rendering only.

**Never copy these hard-coded values into the codebase.**

Always consult the color translation table in `.agents/ui.md` to find the correct CSS variable for any given hard-coded value.
