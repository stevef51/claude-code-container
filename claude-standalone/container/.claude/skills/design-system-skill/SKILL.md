# Design System Skill

This skill guides you through working with any project generated by the Agent Orchestration Platform's design system export. The export format is consistent across all projects — only the package name, tokens, and visual theme vary.

---

## Step 1: Orient Yourself (Always Do This First)

Before writing any code, read these files in order:

1. **`.claude/CLAUDE.md`** — project entry point. Contains the product description, tech stack, design package name and install command, and a summary of the prompt chain chapters. Read this first every time.
2. **`.agents/agents.md`** — index of all agent reference docs and their paths.
3. **`.agents/architecture.md`** — project structure, routing conventions, data fetching patterns.
4. **`.agents/ui.md`** — design system package details, mandatory CSS import rules, color token table, component showcase file paths.
5. **`.agents/coding-conventions.md`** — TypeScript conventions, file naming, React patterns, import order.

> **Do not skip these.** The `.agents/` docs contain mandatory rules (e.g. CSS imports, transpilePackages) that will cause build failures if ignored.

## Step 2: Understand the Folder Structure

Every spec export follows this layout:

```
project-root/
├── .claude/
│   └── CLAUDE.md                   # Entry point — read first
├── .agents/
│   ├── agents.md                   # Index of reference docs
│   ├── architecture.md             # Project structure & routing
│   ├── ui.md                       # Design system rules & token table
│   └── coding-conventions.md      # TypeScript & React patterns
├── design-system/
│   ├── application-design/
│   │   └── [variant]/              # Branding guide, reference page designs, tokens
│   └── application-look-and-feel/
│       └── [variant]/
│           └── package/
│               ├── src/
│               │   ├── tokens.json
│               │   ├── tokens.css
│               │   ├── theme.css
│               │   ├── tailwind-preset.css
│               │   ├── images/
│               │   └── components/ # Component showcase HTML files
│               └── reference/
│                   └── pages/      # Reference page HTML files
└── prompts/
    ├── c1-[chapter-name]/
    │   ├── p1-[prompt-name].md
    │   ├── p2-[prompt-name].md
    │   └── ...
    ├── c2-[chapter-name]/
    └── ...
```

The package name and variant name differ per project — always read `CLAUDE.md` to discover them.

## Step 3: Execute the Prompt Chain

Prompts are in `prompts/` and must be executed in strict order:

**Chapter ordering:** Sort chapter folders by their `c[N]` prefix ascending (c1 → c2 → c3...).  
**Prompt ordering:** Within each chapter, sort prompt files by their `p[N]` prefix ascending (p1 → p2 → p3...).  
**Execution:** Each prompt is a self-contained instruction file — read it fully, then execute it against the codebase before moving to the next.

Do not skip prompts or execute chapters out of order. Each prompt assumes the previous one is complete.

Each prompt file contains:
- **Objective** — what this step accomplishes
- **Context** — relevant background
- **Constraints** — rules to follow
- **Acceptance criteria** — how to verify completion

## Step 4: Design Package Rules (Critical)

Read `.agents/ui.md` for the full authoritative rules. The mandatory rules that apply to every project:

### Install the package
The package name is in `CLAUDE.md`. It is always a `file:` dependency pointing to the package inside `design-system/`. Run `npm install` if not already installed.

### CSS imports — always three lines
```css
@import "tailwindcss";
@import "@design/[package-name]/tailwind-preset.css";
@import "@design/[package-name]/theme.css";
```
Replace `[package-name]` with the actual package name from `CLAUDE.md`. These go in the project's main CSS file (`globals.css` or `index.css`).

### next.config.ts — always required for local packages
```typescript
const nextConfig: NextConfig = {
  transpilePackages: ['@design/[package-name]'],
  // ...other config
};
```
Without this, Next.js will throw `Module not found: Can't resolve 'tailwindcss'` at build time.

### Use CSS variables — never hard-code colors
Always use `hsl(var(--token-name))` syntax. The color token table is in `.agents/ui.md`.  
The component showcase HTML files may contain hard-coded hex/hsl values for standalone rendering — **do not copy these into the codebase**. Use the CSS variable equivalent.

### Reference files — use for structure, not values
- `design-system/.../package/src/components/*.html` — component showcase files. Use for visual/structural reference when building components.
- `design-system/.../package/reference/pages/*.html` — full page designs. Use to understand intended layouts.

## Step 5: Tech Stack Quick Reference

All projects generated by the platform use this stack unless `CLAUDE.md` says otherwise:

| Concern | Technology |
|---------|-----------|
| Framework | Next.js App Router (server components by default) |
| Language | TypeScript strict mode |
| Styling | Tailwind CSS v4 |
| Components | ShadCN/UI (Radix primitives) via `npx shadcn@latest add` |
| Server state | React Query v5 (TanStack) |
| Validation | Zod — all API responses validated before use |
| Runtime schema | Types inferred from Zod: `type X = z.infer<typeof XSchema>` |

**Client components:** Only add `"use client"` when interactivity is genuinely needed. Default to server components.

**Data fetching:** Server components fetch directly. Client components use custom React Query hooks from `lib/hooks/`. All API calls go through service functions in `lib/services/`.

## Step 6: Coding Conventions Quick Reference

Full details in `.agents/coding-conventions.md`. Key rules:

- **File naming:** components → `kebab-case.tsx`, hooks → `use-[name].ts`, services → `[name].service.ts`, schemas → `lib/schemas/[name].ts`
- **No `any` types.** Explicit return types on all exported functions.
- **Prefer `interface`** for object shapes, `type` for unions/intersections.
- **Named exports** preferred over default exports.
- **Import order:** React/Next.js → third-party → internal absolute (`@/`) → relative → type-only imports last.

## Common Pitfalls

| Problem | Cause | Fix |
|---------|-------|-----|
| `Module not found: Can't resolve 'tailwindcss'` | Missing `transpilePackages` in `next.config.ts` | Add `transpilePackages: ['@design/[package-name]']` |
| Colors don't match design | Hard-coded hex values copied from showcase HTML | Replace with `hsl(var(--token-name))` from `.agents/ui.md` |
| Design tokens not applying | Wrong or missing CSS imports | Ensure all three `@import` lines are in main CSS file in correct order |
| Build works locally, fails in Docker | Package not found in container | Ensure `npm install` runs after copying the `packages/` directory |
| Prompts executed out of order | Chapter/prompt folders sorted alphabetically by name | Always sort by `c[N]`/`p[N]` prefix numerically |
